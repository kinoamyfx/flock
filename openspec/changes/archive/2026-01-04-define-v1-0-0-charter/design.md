## Context
项目：Arcadia（中文名“桃源牧歌”）。玩法定位：种田 + 轻度 P2P（以秘境合作为主，秘境内允许 PvP 杀人夺宝），2D 俯视（灯光/迷雾酷炫表现），PC 优先，自研 headless 模拟内核，可用 Godot 做渲染层。

强约束（已拍板）：
- 服务器权威（最大限度防作弊）。
- 断线：角色留在原地 60 秒，可被击杀并触发掉落。
- 重连：秘境未重置原地恢复；秘境已重置入口恢复。
- 死亡掉落：携带物全掉落；安全箱（介子袋）不掉落。
- 安全箱：介子袋 9 格，材料制作；允许在秘境内存取整理。
- 掉落拾取保护：死亡后 10 秒内仅击杀者队伍可捡取。
- 秘境重置：无人后 10 分钟或通关后 10 分钟。
- 撤离/退出：允许撤离点（传送阵），长读条 + 可被打断 + 高成本。
- 经济：交易税 + 手续费；系统销毁作为回收池。
- Mod：MVP 先表现层，后续扩展逻辑层；Mod 本身不允许联网。
- 秘境并发：对外“不限人数”，工程实现为自动分线；单线软上限 64。

## v1.0.0 Version Charter

### Version
`v1.0.0`

### Problem
在“秘境夺宝 + 全掉落”的高风险规则下，作弊与复制会被放大为经济崩溃与玩家信任危机。v1.0.0 必须先建立“权威判定 + 可审计 + 可扩容”的底座，并交付一个可玩闭环（单人经营→准备→入秘境→撤离/死亡→结算→交易/成长）。

### Persona / Use Cases
- 单人经营玩家：在领地进行生产/加工/制造，打造工具/法宝/装备，提升战力与效率。
- 门派经营者：招募小人（成员）分工协作，承担持续维护成本（资源回收点）。
- 秘境玩家（主要）：组队合作刷秘境，获取高价值材料/道具；可在秘境内 PvP 夺宝。

### KPIs / SLIs（首发可观测口径）
- 公平性/反作弊：
  - SLI：鉴权失败/异常断线/重复拾取等风险事件的审计覆盖率 ≥ 99%
  - SLI：Kill → Drop → Loot 事件链可复盘成功率 ≥ 99%
- 稳定性/性能：
  - SLI：单线 64 人时 tick overrun（>1 tick interval）比例 ≤ 1%
  - SLI：单线 64 人时平均端到端输入延迟（P50/P95）可观测（先有指标，再调优）
- 经济健康：
  - KPI：税/手续费销毁量与流通量可观测；NPC 动态兑换可对冲极端资源过剩/短缺趋势（先做“稳定器”，数值后迭代）

### Feature Matrix
#### Must（必须交付）
- 服务器权威秘境（Zone Server）：
  - 战斗/死亡/掉落/背包变更/撤离结算在服务端判定；客户端仅输入与表现。
- 连接与身份可信：
  - Gateway 签发 token（含 key rotation）；Zone 仅验证并绑定身份；鉴权失败带短错误提示。
- 秘境并发与分线：
  - 同图自动分线；单线软上限 64；可路由同队到同线（容量允许）。
- 全掉落规则闭环：
  - 死亡携带物全掉落；安全箱不掉落。
  - 掉落后 10 秒仅击杀者队伍可拾取。
  - 断线留身 60 秒可被击杀并掉落；重连语义按 resetVersion 决策。
- 撤离点（传送阵）：
  - 长读条 + 可被打断 + 高成本；成功后做权威结算（带回战利品）。
- 秘境重置：
  - 无人后 10 分钟 或 通关后 10 分钟自动重置（清理临时实体并把存活玩家回入口）。
- 经济回收机制：
  - NPC 动态兑换（供需调节）；交易税与手续费系统销毁。
- 审计可追溯：
  - 必须能完整还原 Kill → Drop → Loot 链路（含位置、角色、物品 id、时间）。
- 2D 俯视表现底线（Godot 渲染层）：
  - 支持灯光/迷雾；表现不可影响权威状态。
- UI 画风（Art Direction）与体验底线：
  - 必须交付统一的 UI 画风规范（颜色/字体/描边/图标/材质/动效节奏）并在关键界面落地。
  - UI 与 VFX 必须“酷炫且一致”，避免系统 UI 与秘境战斗表现割裂。
- Mod 能力（MVP）：
  - 表现层/资源覆盖，优先级可替换；禁止 Mod 联网。

#### Should（建议交付）
- AOI 与同步底线：为 64 人/线动作战斗提供最小可用的同步与带宽控制。
- 基础反滥用：输入速率限制、技能节奏 sanity check、简单重放保护（sequence）。

#### Could（可后置）
- 门派/寿命/心境深度数值与完整生产链内容库。
- 逻辑层 Mod（含服务端沙箱与签名/白名单策略）。

### Non-goals（明确不做）
- 大规模 MMO（单实例数百人同屏）与跨服经济。
- 完整数值体系与所有内容库（怪物/掉落表/副本库），只保证系统边界与可扩展性。
- 复杂反外挂内核（先把权威与审计闭环做扎实，再迭代对抗策略）。

### Acceptance（验收口径）
- 工程验收：
  - `dotnet build Arcadia.sln` Success
  - `dotnet test Arcadia.sln` Success
  - `bash scripts/smoke_enet.sh` Success（包含 auth 正向 + invalid token 负向）
- 玩法规则验收（可复现）：
  - 断线留身 60 秒可被击杀并触发掉落（安全箱不掉落）。
  - 死亡后 10 秒拾取保护（仅击杀者队伍可捡）。
  - 撤离点长读条可被打断并产生高成本，成功后权威结算。
  - 审计日志可复盘一次 PvP：Kill → Drop → Loot（含物品唯一 id）。
- UI 画风验收（可复现）：
  - 至少完成以下界面并满足统一画风：主菜单、设置、背包（携带/安全箱/整理）、秘境 HUD（血量/灵力/技能/撤离读条）、掉落拾取提示。
  - 至少提供一套 UI Style Guide（文档或可视化资产清单），包含：主色/辅色/强调色、字体、图标风格、面板材质、动效规范（打开/关闭/提示/受击）与可访问性约束（对比度/字号）。
