# Change: MVP 秘境 Zone Server（权威反作弊 + 全掉落 + 64人软上限/自动分线）

## Why
桃源牧歌的“秘境合作为主、允许杀人夺宝、死亡全掉落”会把作弊与复制的破坏性放大到经济系统与玩家留存层面；必须在 MVP 阶段先建立“权威判定 + 可扩展分线”的最小可靠架构边界。

## What Changes
- 新增“秘境 Zone Server”能力：秘境地图逻辑同一张，但按负载自动分线（单线软上限 64 人）。
- 新增“服务器权威判定”边界：战斗/掉落/背包/结算由服务端权威计算与落库，客户端仅上报输入与意图。
- 新增“断线留身可被击杀并全掉落”规则，覆盖断线、崩溃、主动拔线等逃逸场景。
- 新增“区块激活 Tick（Minecraft 类似）+ 世界锚”规则：默认仅加载区块推进；世界锚允许以资源成本在无人时继续推进指定区块。
- 明确技术路线：自研 Headless 模拟内核 + Godot 作为渲染/输入层（PC 优先）。

## MVP Acceptance Criteria
- Zone Server 对秘境战斗/死亡/掉落/背包变更/结算具备权威性，客户端无法直接篡改结果。
- 秘境支持自动分线，单线并发达到 64 人时新进入者自动路由到其他分线。
- 客户端断线后角色留在原地持续 60 秒，可被击杀并触发“携带全掉落（安全箱除外）”；在秘境未重置时重连可原地恢复控制权。
- 若玩家重连时秘境已重置，玩家 MUST 在入口恢复（不回到原死亡/断线点）。
- 物品具备唯一标识与服务端去重约束，无法通过重连/回滚/并发拾取复制物品。
- 具备可追溯审计链路：可还原 Kill → Drop → Loot 的完整链条。
- 客户端（Godot）至少具备 2D 俯视的探索迷雾与战术迷雾表现能力。
