## Context
v1.0.0 已将“UI 画风”提升为 Must：必须交付统一且酷炫的 UI 风格，并在关键界面落地。

关键界面（v1.0.0 Must）：
- 主菜单
- 设置
- 背包（携带/安全箱/整理）
- 秘境 HUD（血量/灵力/技能/撤离读条）
- 掉落拾取提示

## Goals / Non-Goals
- Goals:
  - 统一画风：颜色、材质、图标、字体、动效节奏一致，形成“桃源牧歌”的品牌辨识度。
  - 酷炫但克制：在 2D 俯视 + 灯光/迷雾下，HUD 与场景表现不打架。
  - 可扩展：后续 DLC/官方 mod 能复用同一套 token 与组件库扩展界面。
- Non-goals:
  - 本变更不要求一次性做完全部美术内容库（只保证框架、规范与关键界面）。
  - 本变更不改变权威边界（UI 仍是表现层，不得影响服务端判定）。

## UI Style Guide（交付物定义）
必须至少包含：
- Color Tokens：主色/辅色/强调色/危险色/成功色/中性色；明暗两套（为夜晚/洞窟/迷雾场景准备）
- Typography：字体选择、字号层级、字重与描边/阴影规范
- Iconography：线性/面性、圆角、描边粗细、发光与材质风格
- Panels & Materials：面板材质（木/石/玉/灵力科技）、边框与内阴影规则
- Motion：界面打开/关闭、按钮反馈、提示弹出、受击/危险提示、撤离读条的动效节奏（时长/缓动）
- Accessibility：最小字号、对比度底线（尤其是迷雾/夜景）

## Integration Notes (Godot)
- Theme：用统一 Theme/StyleBox 做全局皮肤，禁止各界面私自硬编码颜色与字体。
- VFX：HUD 提示与场景特效共享色彩系统（避免割裂）。
- Performance：关键界面必须在 60 FPS 目标下可用（PC），并有可执行 profiling checklist。

## Mod Policy (MVP)
- 允许表现层 mod 替换 UI 资源（按优先级覆盖），并记录覆盖审计。
- 禁止 mod 自身联网；任何网络访问必须被阻断并产生日志。
